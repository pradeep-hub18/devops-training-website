Module 01

Linux and shell foundation

Build the operating-system confidence needed for servers, CI agents, containers, logs, networking, and production debugging.

  • File system, permissions, users, groups, packages, services, systemd, cron, processes, and log locations.
  • Shell commands, pipes, redirects, grep/sed/awk basics, SSH, SCP, environment variables, and shell scripting.
  • Networking checks with curl, netstat/ss, dig, nc, ports, DNS, firewalls, and common server health commands.
  • Day-to-day Ansible automation for package updates, service restarts, user creation, config changes, log cleanup, and server health checks.

Lab: prepare a Linux VM, write shell scripts for log checks, and run Ansible playbooks for daily server automation.

Module 02

Git, GitHub, and team workflow

Learn source-control practices used in real DevOps teams, from clean commits to pull-request based delivery.

  • Repositories, branches, commits, tags, pull requests, merge conflicts, revert, reset, stash, and release flow.
  • Branching strategies, protected branches, code reviews, webhooks, repository secrets, and GitHub project hygiene.
  • How to explain a Git workflow clearly in interviews and avoid common production mistakes.

Lab: create a GitHub repo, raise pull requests, resolve conflicts, and trigger a webhook-driven pipeline.

Module 03

Jenkins CI/CD pipelines

Design Jenkins pipelines that build, test, scan, package, and deploy applications with repeatable automation.

  • Jenkins setup, agents, credentials, plugins, freestyle jobs, declarative pipeline syntax, and shared stages.
  • Build artifacts, test reports, quality gates, Docker build stages, deployment stages, rollback thinking, and notifications.
  • Pipeline troubleshooting for failed builds, missing credentials, agent issues, and broken webhooks.

Lab: create a Jenkinsfile that builds an app, creates a Docker image, and deploys it to Kubernetes.

Module 04

Docker containers and registries

Containerize applications properly and understand how images move from developer machines to production clusters.

  • Images, containers, Dockerfiles, layers, tags, volumes, networks, Compose, logs, exec, and resource limits.
  • Image optimization, multi-stage builds, private registries, ECR basics, security scanning, and secret handling.
  • Debugging failed containers, port mapping issues, missing environment variables, and broken image builds.

Lab: dockerize a sample app, push it to a registry, and run it with environment-specific configuration.

Module 05

Kubernetes core concepts

Understand the Kubernetes objects and operational patterns needed to deploy and support containerized applications.

  • Pods, ReplicaSets, Deployments, Services, namespaces, labels, selectors, ConfigMaps, Secrets, and probes.
  • Resource requests and limits, rolling updates, rollbacks, node scheduling, kubectl commands, and YAML structure.
  • Debugging pod events, logs, describe output, service discovery, readiness failures, and pending workloads.

Lab: deploy a multi-service application with config, secrets, health checks, service access, and rollout history.

Module 06

Helm packaging and releases

Convert Kubernetes manifests into reusable Helm charts that support clean releases across dev, test, and production.

  • Chart structure, values files, templates, helpers, release names, upgrades, rollbacks, and environment overrides.
  • Reusable chart patterns for deployments, services, ingress, config maps, secrets, probes, and image tags.
  • Common Helm mistakes such as bad indentation, missing values, failed upgrades, and drift from manual changes.

Lab: package the Kubernetes app into a Helm chart and deploy separate dev and prod-style values.

Module 07

ArgoCD GitOps delivery

Move from manual deployment to GitOps, where Git becomes the source of truth for application releases.

  • ArgoCD installation, applications, repositories, sync policies, app health, desired state, and drift detection.
  • Helm with ArgoCD, auto sync, manual sync, rollback, multi-environment branches, and promotion workflows.
  • How to troubleshoot sync failures, unhealthy resources, repo access issues, and out-of-sync applications.

Lab: deploy the Helm chart through ArgoCD and demonstrate Git-based rollback and promotion.

Module 08

Terraform infrastructure as code

Provision cloud infrastructure with reusable code, safe reviews, predictable state, and practical module design.

  • Providers, resources, variables, outputs, locals, state, remote backends, workspaces, modules, and plan/apply flow.
  • AWS VPC, subnets, security groups, IAM basics, ECR, EKS dependencies, and environment-specific modules.
  • Real-world safeguards: state locking, drift review, import basics, plan review, and avoiding destructive changes.

Lab: create Terraform modules for AWS network and deployment prerequisites used by the EKS project.

Module 09

AWS cloud services for DevOps

Learn the AWS building blocks that DevOps engineers use while deploying, securing, and operating cloud workloads.

  • IAM users, roles, policies, VPC, subnets, route tables, security groups, EC2, ECR, load balancers, and Route 53 basics.
  • Cloud deployment patterns, least-privilege access, image registry flow, networking dependencies, and cost-aware setup.
  • Common AWS interview scenarios around IAM access, VPC connectivity, ECR authentication, and load balancer issues.

Lab: prepare AWS infrastructure and registry flow for a containerized application release.

Module 10

AWS EKS production deployment

Bring Kubernetes and AWS together by deploying workloads to EKS with real cloud networking and delivery practices.

  • EKS cluster access, node groups, kubectl authentication, namespaces, ECR image pulls, service accounts, and load balancers.
  • AWS Load Balancer Controller concepts, ingress flow, DNS mapping, rollout validation, and production-style release checks.
  • Troubleshooting node readiness, IAM permissions, image pull errors, service exposure, and ingress target health.

Lab: deploy the final application to EKS using Docker, Helm, ArgoCD, ECR, and AWS load balancing.

Module 11

Monitoring, logs, and alerts

Understand observability basics so you can detect issues, read symptoms, and explain production health clearly.

  • Metrics, logs, dashboards, alerts, service health, application latency, resource saturation, and failure symptoms.
  • Prometheus and Grafana concepts, Kubernetes metrics, pod logs, event review, and alert investigation workflow.
  • How to explain monitoring in interviews: what to watch, why it matters, and what action follows an alert.

Lab: build dashboards for Kubernetes workloads and practice incident-style log and metric analysis.

404 Module 12

DevOps troubleshooting labs

Practice realistic failures so you can debug faster in projects and answer scenario-based interview questions.

  • ImagePullBackOff, CrashLoopBackOff, pending pods, failed probes, bad secrets, service mismatch, and DNS failures.
  • Broken Jenkins pipelines, failed Helm upgrades, ArgoCD sync errors, Terraform plan failures, and AWS IAM mistakes.
  • Structured debugging: symptom, scope, logs, events, config, recent change, fix, validation, and prevention.

Lab: solve guided break/fix scenarios across Jenkins, Docker, Kubernetes, Helm, ArgoCD, AWS, and EKS.

HR Module 13

Resume and interview preparation

Turn the course project into a confident interview story with architecture, decisions, failures, and measurable outcomes.

  • Resume project framing, GitHub portfolio cleanup, architecture explanation, and DevOps responsibility mapping.
  • Mock interviews, scenario questions, command questions, cloud questions, troubleshooting questions, and HR preparation.
  • How to explain CI/CD, Kubernetes, EKS, Terraform, monitoring, and incidents without memorized answers.

Lab: prepare a final project walkthrough, interview Q&A sheet, and resume-ready DevOps project summary.

Module 14

Python basic to advanced for DevOps

Learn Python from fundamentals to automation scripts that help with files, APIs, cloud tasks, reporting, and operational workflows.

  • Python setup, variables, data types, conditions, loops, functions, modules, virtual environments, and clean script structure.
  • File handling, CSV/JSON parsing, error handling, logging, regular expressions, command-line arguments, and reusable utilities.
  • DevOps automation with subprocess, HTTP API calls, server health checks, log analysis, report generation, and basic AWS/Boto3 concepts.

Lab: build Python scripts for server health reports, log filtering, API checks, and daily DevOps automation tasks.
